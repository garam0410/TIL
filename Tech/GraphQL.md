# **GraphQL**

    1. 페이스북에서 만든 쿼리언어

    2. 기존 서비스의 API를 구현할 떄는 REST API를 사용

    3. REST API를 사용하여 API를 구현 한다면, Client에서 어떠한 기능이 필요할 때마다 새로 만들어줘야 함

    4. 서비스 규모가 커지면 수많은 endpoint가 생기게 됨

    5. Client에서 쿼리를 만들어서 서버로 보냈을 때, 그 결과에 맞게 반환해주면 더 좋음

    6. GraphQL이 이를 가능하게 해주는 것

    7. 프로토콜의 제약이 없어서 WebSocket이나 MQTT 위에서 사용 가능

    8. 데이터 베이스도 상관 없음

    9. 이미 구현된 시스템에 도입해도 기존에 있던 시스템이 무너지지 않아서 부담없이 적용 가능
